# PySide Test Disposition

## Status Legend

- `candidate-accept`: keep and trace to requirement IDs in the coverage matrix
- `review-before-accept`: useful, but baseline assumptions must be checked before acceptance
- `rewrite-or-augment`: keep only as input until compatibility or release-path rules are settled
- `accept-for-m4a`: target file should become direct proof for the M4a filtering parity slice
- `rewrite-for-m4a`: target file stays in scope for M4a but must be rewritten or extended before it counts as direct proof
- `accept-for-slice`: target file should become direct proof for the M5 object-detection reference slice
- `rewrite-for-slice`: target file stays in scope for M5 but must be rewritten to match the approved compatibility and runtime-state proof
- `defer-beyond-slice`: file remains valuable, but the M5 slice will not make it proof-grade yet
